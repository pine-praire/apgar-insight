## Текущее состояние

Хорошая новость: код админ-раздела уже полностью написан в проекте:
- `src/routes/admin.tsx` — страница `/admin` с таблицей пользователей, статистикой (всего пользователей, тестов, средний балл) и раскрывающейся историей тестов по каждому пользователю
- `src/lib/auth.tsx` — уже подгружает флаг `isAdmin` из таблицы `user_roles`
- `src/routes/dashboard.tsx` — уже показывает кнопку-щит для перехода в админку, если пользователь админ
- Файл миграции `supabase/migrations/20260504120000_admin_roles.sql` — описывает таблицу `user_roles`, RLS-политики и функцию `get_admin_users()`

Проблема: эта миграция **не применена** к базе данных. В БД сейчас существуют только таблицы `profiles` и `apgar_results`, таблицы `user_roles` нет, функции `get_admin_users` тоже нет. Поэтому страница `/admin` сейчас не работает.

## Что нужно сделать

### 1. Применить миграцию ролей к БД
Создать таблицу `public.user_roles` с RLS:
- `user_id` (PK, FK на `auth.users`, каскадное удаление)
- `role` (text, проверка `= 'admin'`)
- `created_at`

Политики:
- SELECT: пользователь видит только свою роль (нужно для показа кнопки админа в UI)
- INSERT/UPDATE/DELETE: запрещены через клиент — роли назначаются только напрямую через SQL (service-role)

Дополнительные RLS-политики (additive):
- `profiles`: админ может читать все профили
- `apgar_results`: админ может читать все результаты

Функция `public.get_admin_users()`:
- `SECURITY DEFINER`, `search_path = public, auth`
- Внутри проверяет, что вызывающий — админ (defence-in-depth)
- Возвращает `id, email, display_name, created_at` через JOIN `auth.users` + `public.profiles`
- EXECUTE доступен только роли `authenticated`

### 2. Назначить вас администратором
После применения миграции добавить вашу запись в `user_roles`. Для этого мне нужен ваш email (тот, которым вы зарегистрировались на сайте). После того как вы его сообщите, я выполню вставку:
```sql
INSERT INTO public.user_roles (user_id, role)
SELECT id, 'admin' FROM auth.users WHERE email = '<ваш email>';
```

### 3. Проверка
- Зайти на сайт под вашим аккаунтом
- На дашборде должна появиться иконка-щит рядом с кнопкой выхода
- Клик ведёт на `/admin` со списком всех пользователей и их результатов
- Не-админы при попытке открыть `/admin` редиректятся на дашборд

## Технические детали

- Никаких изменений в коде React не требуется — он уже готов
- Изменения только в БД: одна миграция + одна вставка с вашим email
- Безопасность: роли хранятся в отдельной таблице (не в `profiles`), проверяются на сервере через RLS и функцию `SECURITY DEFINER`, клиент не может назначить себе роль admin
- `SUPABASE_SERVICE_ROLE_KEY` для записи в `user_roles` не используется в клиентском коде — назначение админов происходит только через прямой SQL-доступ
